# 犬猫の健康管理アプリ

保護団体内の複数人でログインして使う、犬猫の健康管理Webアプリです。
GitHub Pages(画面の公開)+ Firebase(ログイン・データ保存)で動きます。

## 団体名の表示・他団体での使い回しについて
`site-config.js` の `ORG_NAME` を書き換えるだけで、ログイン画面・メイン画面に表示される団体名を変更できます(空文字にすると団体名なしの表示になります)。同じファイルの `FACILITY_LABEL` / `FOSTER_LABEL` で、「シェルター(施設)」「個人宅預かり」といった保護場所の呼び方も、団体ごとの言い方(例: 「保護施設」「センター」「一時預かり」など)に変更できます。他の保護団体がこのアプリを使う場合も、基本的にはこの1ファイルと、それぞれの団体用のFirebaseプロジェクト設定(`firebase-config.js`)を用意するだけで使い回せます。

## 譲渡済みの管理について
- 猫の詳細画面から「譲渡済みにする」を押すと、一覧画面ではデフォルトで非表示になります(「譲渡済みの子も表示する」にチェックを入れれば見られます)。
- 記録を完全に消したい場合は、管理者・責任者だけが押せる「この子のデータを完全削除」ボタンで、日々の記録・医療記録もまとめて削除できます(元に戻せないため、確認が2回出ます)。

## 日々の記録と投薬管理について
- 日々の記録は「早朝・朝・昼・夕方・夜・深夜」から時間帯を選んで記録します(世話をした時刻も任意で記録可能)。シェルターは普段通り「朝・夜」の2回でOKですが、生後間もない子猫がいる預り宅など、1日5回前後の頻繁な給餌が必要な場合も、細かい時間帯を選んで記録できます。
- 医療記録で「投薬」を選ぶと、**投与方法(飲み薬・塗布・点眼・点耳・注射・その他)**、投薬するタイミング(早朝〜深夜)、1回あたりの分量を登録できます。
- 投薬を登録しておくと、該当する時間帯で日々の記録を追加する際に「投薬チェック」が自動で表示され、投与したかどうかをその場でチェックできます。
- 投薬終了予定日を設定すると、その日以降は日々の記録のチェックに出てこなくなります。

## 役割(権限)の管理について

このアプリは、役割によって見られるデータが変わります。

- **管理者・責任者**: すべての犬猫のデータを見られる
- **施設メンバー**: 保護場所が「シェルター(施設)」の犬猫だけ見られる
- **預り先**: 自分が担当している(割り当てられた)犬猫だけ見られる

### 新しく登録した人に役割を割り当てる手順
1. 新しくユーザー登録してもらう(役割は最初「未設定」になり、この状態だとデータは何も見えません)
2. Firebaseコンソール →「Firestore Database」→「データ」タブを開く
3. `users` というコレクションの中に、その人のドキュメント(ユーザー名などが入っている)があるので開く
4. `role` フィールドの値を、以下のいずれかに書き換えて保存する
   - `管理者`
   - `責任者`
   - `施設メンバー`
   - `預り先`

### 個人宅預かりの犬猫を登録するとき
犬猫の登録画面で「保護場所」を「個人宅預かり」にすると、「預かり担当者」を選ぶ欄が出てきます。ここで選んだ人だけが、その犬猫のデータを見られるようになります。**選択肢に出てくるのは、あらかじめ役割が「預り先」に設定されている人だけ**なので、先にその人の役割設定を済ませておく必要があります。

## セットアップ手順

### 1. Firebaseプロジェクトを作る
1. https://console.firebase.google.com/ でプロジェクトを作成
2. 「Authentication」→「Sign-in method」→「メール/パスワード」を有効化
3. 「Firestore Database」→ データベースを作成(リージョンは `asia-northeast1` 推奨)
4. 「Firestore Database」→「ルール」タブ → このフォルダの `firestore.rules` の内容を貼り付けて公開
5. 「プロジェクトの設定」→「マイアプリ」→ ウェブアプリを追加 → 表示される設定値をコピー

### 2. 設定を貼り付ける
`firebase-config.js` を開き、`firebaseConfig` の中身をFirebaseコンソールで表示された値に置き換える。

### 3. GitHubにアップロードしてGitHub Pagesで公開
1. GitHubリポジトリにこのフォルダの全ファイルをアップロード
2. リポジトリの「Settings」→「Pages」→ ブランチを選んで公開
3. 公開されたURLの `index.html` にアクセスすればログイン画面が表示される

### 4. 使い方
- 初回は「ユーザー名(半角英数字)」または「メールアドレス」とパスワードで「新規登録」する(メンバーごとに個別のアカウントを作ってください。役割の割り当てが個人単位のため、1つのアカウントの共有はおすすめしません)
- ログイン後、右下の「+」から犬猫を登録(登録時に犬/猫を選べます)
- 登録した動物をタップすると「日々の記録」「医療記録」を追加できる
- 団体の全メンバーが同じデータを見られる(ログインさえしていれば誰でも閲覧・編集可能)

## 壁紙のカスタマイズについて
アプリ画面(ログイン後)の右上にある「🎨 壁紙」ボタンから、好きなものを選べます。選んだ内容はアカウントごとにFirestoreへ保存されるので、他の人の画面には影響せず、別の端末からログインしても自分が選んだ壁紙が反映されます。
(ログイン前の画面は、まだ誰のアカウントか分からないため、共通の背景のままになります)

**選べる壁紙(4種類):**
- **イラスト**: 肉球柄のさりげないイラスト(最初から使えます)
- **写真**: 犬猫の写真(下記の手順で `wallpaper-pet.jpg` を追加すると使えるようになります)
- **こちらで用意する写真**: 団体共通の写真(下記の手順で `wallpaper-common.jpg` を追加すると使えます)。**新しく登録した人は、最初はこの壁紙が選ばれた状態になります**
- **自分で写真をアップロード**: メンバーそれぞれが自分の端末から好きな写真を選べます。自動で圧縮され、その人のアカウントだけに表示されます

**写真プリセットを追加する手順:**
1. リポジトリの中に `assets` という名前のフォルダを作る
2. その中に、以下の名前で画像ファイルをアップロードする
   - `wallpaper-pet.jpg`(犬猫の写真)
   - `wallpaper-common.jpg`(団体共通で使う写真)
3. GitHubに反映すれば、壁紙メニューにその写真が表示されるようになります

## 使い方ガイドについて
ログイン画面・メイン画面の両方から「使い方ガイド」というリンクで、操作方法をまとめたページ(`guide.html`)を開けます。使い方が分からないメンバーには、まずこのページを見てもらうよう案内してください。

## ファイル構成
- `index.html` / `auth.js` … ログイン・新規登録画面
- `guide.html` … 使い方ガイドページ
- `app.html` / `app.js` … 犬猫の一覧・詳細・記録画面
- `firebase-config.js` … Firebaseの接続設定(自分の値に書き換える)
- `firestore.rules` … Firestoreのセキュリティルール(Firebaseコンソールに貼り付ける)
- `style.css` … デザイン
